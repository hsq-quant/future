# INE 原油新闻情绪项目汇报讲稿

## 1. 开场

大家好，我们这次做的是一个“新闻情绪预测期货收益”的项目。

简单说，我们想复刻一篇论文的核心思路，但把研究对象从海外原油期货迁移到中国上海国际能源交易中心的原油期货 `SC`。

这次项目最后收成了两条主线：

- 分类主线：新闻情绪能不能预测下周涨跌
- 回归主线：新闻情绪能不能形成更像量化因子的连续信号

最后汇报时，我们不需要再临时拼图，直接按 workbook 底稿顺序讲：

- 方法对比
- 11 个特征
- 新闻覆盖
- IC 分布
- 模型族对比
- 市场走势
- 累计收益
- 成本/滑点
- 仓位分布
- Regime 分阶段

## 2. 整体思路

这个项目可以理解成一条很清晰的链：

- 第一步，收集历史新闻
- 第二步，用大模型把新闻转成结构化情绪分数
- 第三步，按周聚合成 11 个因子
- 第四步，用这些因子预测下一周的市场变化
- 第五步，再把预测接成不同风格的交易执行

所以它不是单纯做新闻抓取，也不是单纯做机器学习，而是一条从新闻到因子、从因子到预测、再从预测到交易检验的完整流程。

## 3. 市场数据部分

在市场数据这边，我们用的是 `INE SC` 不同月份合约的日线数据。

期货合约会到期，所以不能直接拿某一个合约一路往后看，也不能把不同月份的收盘价直接拼起来。

因此我们先构建了连续价格序列。规则是：

- 默认跟踪近月合约
- 在最后交易日前 `5` 个交易日滚到下一合约
- 滚动时做价差调整，消除换月价格台阶

然后把日线变成周线，用本周新闻去预测下一周。

## 4. 新闻数据部分

最终我们把新闻输入收敛成两条主线：

- `V1`：中文新闻主链
- `V3`：英文全球能源新闻主链

这里最关键的汇报口径是：

我们最后统一采用 **aligned 窗口** 来比较所有结果：

- `2021-01-22`
- 到 `2025-09-05`

这样做的好处是：

- `V1` 和 `V3` 在同一个交易窗口里比较
- 分类和回归也能放在同一张表里解释
- 不会把后段不重合的收益混进去

## 5. 大模型打分

抓到新闻后，我们让大模型按照论文定义，对每篇新闻打 5 个维度：

- `relevance`
- `polarity`
- `intensity`
- `uncertainty`
- `forwardness`

最重要的一条规则是：

- 如果 `relevance < 0.1`
- 其他四个维度全部置空

也就是说，如果新闻和原油几乎没关系，就不强行给它打利多利空分数。

## 6. 因子构建

打分之后，我们把同一周的文章聚合成 11 个周频特征。

最直观可以这样讲：

- 有多少篇新闻
- 新闻整体偏利多还是利空
- 情绪有多强
- 不确定性高不高
- 和上一周相比变化大不大

其中最关键的一点是：

- 和原油更相关的新闻，权重更高

## 7. 模型与验证

模型使用 `LightGBM`，验证方式不是普通随机切分，而是：

- `5-fold`
- `expanding window`

也就是每一折都只能用过去训练，再去预测更后面的数据。

这次我们最后跑了两条模型线：

- 分类：
  - 输出下周上涨概率
  - 指标看 `AUC`、`Accuracy`、`IC`
- 回归：
  - 直接预测下一周收益率
  - 指标重点看 `IC`

## 8. 当前最重要的结果

### 8.1 分类结果

`V1 Chinese classification`

- `AUC = 0.5366`
- `Accuracy = 0.5435`
- `IC = 0.0614`

`V3 English classification`

- `AUC = 0.5810`
- `Accuracy = 0.5515`
- `IC = 0.0950`

如果只看模型指标，`V3` 分类更强。

但交易层不是这样。

在共同窗口下：

- `V1 classification / always_in`
  - 累计收益 `155.07%`
- `V1 classification / long_only`
  - 累计收益 `210.96%`

- `V3 classification / always_in`
  - 累计收益 `-28.69%`
- `V3 classification / long_only`
  - 累计收益 `60.04%`

所以这里最值得讲的一句话是：

**模型指标更高，不等于交易结果就更好。**

`V3` 分类的问题不是模型完全没用，而是“预测跌就直接做空”的执行太差。

### 8.2 回归结果

`V1 Chinese regression`

- `IC = -0.0673`
- 因子本身较弱

`V3 English regression`

- `IC = 0.1186`
- 平均方向准确率 `53.08%`

然后我们给回归信号接了几种更合理的执行方式。

最好的版本是：

- `V3 regression / long_only_factor`
  - 累计收益 `96.65%`
  - 夏普 `0.9010`

所以如果要讲“更贴近量化因子扩展”的主结果，最适合讲这一版。

## 9. 分阶段压力测试

为了看策略是不是只在某个阶段有效，我们把共同窗口再拆成了 5 段：

- `Recovery Bull`
- `War Spike`
- `Post-Spike Bear`
- `OPEC-Supported Range`
- `Oversupply Bear`

这里最值得讲的结论是：

1. `V1 classification / long_only`
   - 是分类里最稳的一版

2. `V3 classification`
   - 模型指标好
   - 但交易执行明显弱

3. `V3 regression / long_only_factor`
   - 是当前最平衡、最适合继续扩展的一版

4. `War Spike`
   - 是所有策略都比较难做的压力阶段

## 10. 最后结论

最后我会把结论收成 4 句话：

1. 我们成功把论文的新闻情绪框架迁移到了 `INE SC`。
2. 中文新闻主链能形成弱但真实的预测力。
3. 英文 `V3` 在模型层更强，但分类执行层不够好。
4. 如果继续往量化方向走，最值得保留的是：
   - 分类主线做直观展示
   - 回归因子主线做实盘扩展
