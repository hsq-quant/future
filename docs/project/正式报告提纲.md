# 基于大语言模型新闻情绪因子的原油期货收益预测研究

## 正式报告提纲

### 1. 研究背景与核心问题

本报告围绕一个非常具体的问题展开：新闻情绪能否帮助预测原油期货收益，尤其是大语言模型抽取出的多维新闻情绪信号，是否能够迁移到中国原油期货市场，并进一步形成具有交易意义的量化因子。

原油市场天然受以下因素影响：

- 供需变化
- OPEC 与主要产油国政策
- 地缘政治冲击
- 库存变化
- 宏观风险偏好

因此，新闻信息在原油市场中往往比在很多普通资产中更重要。传统新闻情绪研究通常只使用单一 polarity，即把新闻粗略地划分为“利多/利空”，但这种方式容易忽略新闻强度、不确定性和未来导向等更复杂的信息。原论文的价值就在于：不再把新闻压缩成单维信号，而是使用大语言模型抽取多维度情绪结构。

本项目的三个核心问题是：

1. 原论文的多维 LLM 新闻因子方法，能否迁移到中国 INE SC 原油期货市场。
2. 中文新闻和英文全球能源新闻分别能否形成有效的周度预测信号。
3. 当预测进一步转化为交易策略时，分类方法与回归因子方法哪个更有实际价值。

### 3. 原论文策略原理

#### 3.1 数据来源与筛选哲学

原论文使用 **Alpha Vantage News Sentiment API**，并利用其自带的 `energy_transportation` 话题标签提取与原油相关的能源新闻。

这里最关键的地方是：论文的前置筛选非常宽松。

- 它并没有使用大量关键词去硬筛；
- 也没有在打分前人为剔除“看起来不相关”的文章；
- 而是采用“宽入口 + LLM relevance 软过滤”的方式。

论文原始新闻样本：

- 时间：`2020-01-01` 到 `2025-12-31`
- 去重后总量：`29,153` 篇

出于预算和计算成本限制，论文并没有给全部新闻打分，而是采用了**按月分层随机抽样**：

- LLM 抽样集：`8,639` 篇，约为全样本的 `20%`

#### 3.2 多模型、复合模型与交互思路

原论文并不是只用一个大模型。它使用了：

- `GPT-4o`
- `Llama 3.2-3b`
- `FinBERT`
- `AlphaVantage` 自带情绪信号

其中最重要的一点是：

**论文最优结果不是单模型，而是 GPT-4o 与 FinBERT 的组合。**

这说明原论文的方法并不是“让一个 LLM 端到端做完所有事情”，而是强调：

- LLM 提供更细粒度的语义理解；
- 金融专用模型提供传统金融文本情绪的稳健基线；
- 二者存在互补性。

这也是我们和原论文之间最关键的方法差异之一：论文实际上走的是**复合模型、多特征融合**路线，而不是简单单模型路线。

#### 3.3 五维打分

原论文让模型对每篇新闻提取 5 个维度：

1. `relevance`
2. `polarity`
3. `intensity`
4. `uncertainty`
5. `forwardness`

其规则是：

- 先判断 `relevance`
- 当 `relevance < 0.1` 时，其余四维置空
- 但后续特征聚合时，不做额外 relevance 硬过滤，而是把 relevance 当作加权项

这意味着论文的思路不是“预先把弱相关新闻删掉”，而是让模型自己学习软权重。

#### 3.4 特征构建与机器学习

论文把文章级信号按周聚合成 11 个特征：

- `article_count`
- `relevance_mean`
- `polarity_mean`
- `intensity_mean`
- `uncertainty_mean`
- `forwardness_mean`
- `polarity_std`
- `uncertainty_std`
- `polarity_momentum`
- `uncertainty_momentum`
- `forwardness_momentum`

机器学习主线：

- 任务：分类
- 模型：LightGBM Classification
- 标签：下一周收益 `> 0` 与否
- 评估：AUC、Accuracy、IC

原论文最重要的经验结论之一是：

- `intensity` 与 `uncertainty` 相关特征的重要性很高；
- 新闻预测价值不止来自“正负面”，而来自多维度情绪结构；
- 组合模型优于单一模型。

### 6. 标的与新闻源设计

#### 6.1 交易标的：INE SC 原油期货

我们使用的是 **INE SC 连续合约**，通过滚动主力合约方式构造周度价格序列，并进一步构造：

- 周度收益
- 下一周收益
- 下一周涨跌标签

为了避免不同版本样本区间不一致带来的误导，最终正式对比统一使用 **aligned 共同窗口**：

- `2021-01-22`
- 到 `2025-09-05`

#### 6.2 新闻源设计

##### V1：中文主线

核心来源是中文财经新闻，以：

- `NBD`（每日经济新闻）

为主。

V1 的意义：

- 最早打通全链路；
- 与国内市场语境一致；
- 分类策略结果最直观。

##### V2：中英混合

V2 只是中间迭代版本，结论是：

- 中英新闻简单拼接并没有稳定增强预测；
- 多语言并不自动等于更强信息；
- 因此 V2 在最终汇报中只保留简短说明，不作为主结果线。

##### V3：英文主线

英文新闻主要来自：

- Hugging Face 历史金融新闻主源
- `2024` 年补充金融新闻源

V3 经过了多轮工程修正：

- shard 覆盖审计
- 补关键 shard
- 放宽英文筛选逻辑
- 对齐共同窗口

最终形成一条独立可评估的英文主线，用来回答：

**全球英文能源/金融新闻，能否增强对中国 INE 原油期货的预测。**

### 7. V1 与 V3 的版本迭代

#### 7.1 V1：中文单源版本

V1 是最先完成的主线版本：

- 使用中文财经新闻
- 先跑分类
- 再扩展到回归因子

V1 的重要性在于：

- 它不是只是 baseline，而是一个能跑出比较好课堂展示效果的版本；
- 尤其在分类 `long_only` 执行层下，结果最直观，也最适合作为展示主线。

#### 7.2 V3：英文主线版本

V3 的目标不是“增强一个中文模型”，而是把英文新闻单独作为一条候选主线去检验。

V3 的主要挑战是：

- 英文历史新闻覆盖不均衡；
- 2024 年缺口明显；
- 过严筛选会损失大量样本；
- 打分成本和耗时显著高于中文。

V3 的工程价值在于：

- 它让我们从“数据工程问题”中区分出了“模型问题”和“交易问题”；
- 最终发现 V3 的回归因子本身是有效的，但交易映射方式非常关键。

### 8. 打分、特征构建与机器学习流程

#### 8.1 五维打分

所有新闻先用 Qwen 2.5 打五维分：

- `relevance`
- `polarity`
- `intensity`
- `uncertainty`
- `forwardness`

规则完全遵循论文：

- 先判断 `relevance`
- 若低于阈值，其余维度置空
- 不在后续聚合时再做硬 relevance 过滤

#### 8.2 11 个周频特征

新闻按交易周聚合后，形成 11 个特征。  
其中最重要的三类信息是：

- 周内新闻总量：`article_count`
- 周内平均情绪水平：如 `polarity_mean`、`intensity_mean`
- 周度变化与分歧：如 `uncertainty_std`、`polarity_momentum`

这些特征保留了论文的核心思想：

- 数量本身也是信息
- 强度与不确定性不应被丢掉
- 新闻变化趋势比静态极性更重要

#### 8.3 分类与回归双路线

我们最终保留两条训练主线：

##### 分类路线

- 与原论文主方法一致
- 目标：预测下一周涨跌
- 指标：AUC、Accuracy、IC

##### 回归路线

- 我们的主要扩展
- 目标：预测下一周真实收益率
- 指标核心看 IC

回归路线更接近量化因子使用逻辑，因为它保留了收益幅度信息。

### 10. 因子结果：分类与回归的核心比较

这一页是正式汇报的核心之一。  
所有结果都以 aligned 共同窗口为准。

#### 10.1 分类结果

最终 aligned 结果中：

- `V1 Chinese classification`
  - `AUC = 0.5366`
  - `Accuracy = 0.5435`
  - `IC = 0.0614`

- `V3 English classification`
  - `AUC = 0.5810`
  - `Accuracy = 0.5515`
  - `IC = 0.0950`

这说明在**预测指标层面**，V3 分类是优于 V1 分类的。

#### 10.2 回归结果

- `V1 Chinese regression`
  - `IC = -0.0673`
  - `Directional Accuracy = 0.4673`

- `V3 English regression`
  - `IC = 0.1186`
  - `Directional Accuracy = 0.5308`

这说明在回归因子层面：

- V1 的因子较弱，甚至偏反向；
- V3 的回归因子更像一个真正有信息量的量化因子。

#### 10.3 结论

从纯模型视角看：

- `V3 classification` 优于 `V1 classification`
- `V3 regression` 显著优于 `V1 regression`

也就是说，**英文主线在预测层面并不差，甚至更强。**

### 12. 回测结果与交易表现

#### 12.1 分类交易结果

共同窗口下，分类最值得讲的两条是：

- `V1 classification / long_only`
  - 累计收益：`210.96%`
  - 夏普：`1.2149`
  - 相对 benchmark 超额：`+42.24%`

- `V3 classification / long_only`
  - 累计收益：`60.04%`
  - 夏普：`0.4201`
  - 相对 benchmark 超额：`-108.68%`

这说明：

- V1 分类虽然模型指标一般，但在中文单源与 long-only 执行下，形成了非常适合展示的策略；
- V3 分类虽然 AUC/Accuracy 更高，但落地到交易时，收益明显不如 V1。

#### 12.2 回归因子交易结果

回归最重要的两条是：

- `V1 regression / long_only_factor`
  - 累计收益：`64.98%`
  - 夏普：`0.6494`

- `V3 regression / long_only_factor`
  - 累计收益：`96.65%`
  - 夏普：`0.9010`

这说明：

- 回归框架下，V3 显著优于 V1；
- V3 regression 更适合作为“量化因子研究成果”展示；
- 但它仍未跑赢长期做多原油的 benchmark。

#### 12.3 为什么会出现“高 IC 但收益不如 V1 分类”

这是本项目最重要的发现之一：

- 高 IC 反映的是排序能力；
- 但单品种交易收益还取决于：
  - 多空映射是否合理；
  - benchmark 是否过强；
  - 错误信号是否恰好发生在大波动周；
  - 市场是否整体偏强，导致做空腿被持续压制。

因此：

- `V3` 因子可能是更好的“因子”
- 但 `V1 classification / long_only` 可能是更好的“单品种课堂展示策略”

### 13. 牛市 / 熊市 / 震荡市分阶段表现

我们把 aligned 窗口进一步分成多个 regime：

- `Recovery Bull`
- `War Spike`
- `Post-Spike Bear`
- `OPEC-Supported Range`
- `Oversupply Bear`

#### 13.1 分类策略的分阶段特征

- `V1 classification / long_only`
  - `Recovery Bull`：`85.81%`
  - `Post-Spike Bear`：`57.30%`
  - `Oversupply Bear`：`16.64%`

- `V3 classification / long_only`
  - `Recovery Bull`：`89.55%`
  - `Post-Spike Bear`：`-8.80%`
  - `Oversupply Bear`：`-1.84%`

这说明：

- V3 分类更适合部分上涨阶段；
- 但在熊市和再平衡阶段，稳定性明显不如 V1。

#### 13.2 回归策略的分阶段特征

- `V3 regression / long_only_factor`
  - `Recovery Bull`：`64.97%`
  - `War Spike`：`-5.44%`
  - `Post-Spike Bear`：`2.69%`
  - `OPEC-Supported Range`：`16.29%`
  - `Oversupply Bear`：`5.56%`

它比分类更平滑，也更符合“因子在不同市场结构中表现不同”的逻辑。

#### 13.3 分阶段结论

- `War Spike` 是所有版本都比较难做的阶段；
- `V1 classification / long_only` 更像一个“抓趋势的简单策略”；
- `V3 regression / long_only_factor` 更像一个“在多个市场阶段都有一定解释力的研究型因子”。

### 14. 与原论文的对比

#### 14.1 相同点

- 使用新闻驱动方法
- 使用五维情绪评分
- 构建 11 个周频特征
- 使用 LightGBM 做周频预测

#### 14.2 不同点

原论文：

- 标的是 WTI
- 新闻源是 Alpha Vantage
- 采用多模型组合
- 最优模型融合了 GPT-4o 与 FinBERT
- 主要关注预测，不做交易

我们的项目：

- 标的是 INE SC
- 使用中文单源与英文全球新闻两条实验主线
- 使用 Qwen 2.5 做情绪打分
- 增加分类与回归双路线
- 增加交易层、成本层与 regime 层分析

#### 14.3 为什么我们的结果与论文不同

这部分很重要，不能只说“我们弱一些”。

原因可能来自：

1. **新闻源差异**
   论文数据源更稳定、更贴合 WTI 全球市场；
   我们的中文与英文历史新闻源覆盖和风格差异都更大。

2. **模型差异**
   论文最优版本是 GPT-4o + FinBERT 组合；
   我们主要使用 Qwen 2.5 单一打分模型。

3. **市场差异**
   WTI 与全球英文能源新闻天然更匹配；
   INE SC 受国内与全球因素共同影响，市场映射更复杂。

4. **目标差异**
   论文重点是预测；
   我们额外加入交易，因此结果更容易受到执行层影响。

---

### 15. 主要发现与启发

本项目最终形成了几条非常明确的结论。

#### 15.1 中文单源并不一定弱

V1 告诉我们：

- 单一中文新闻源虽然简单；
- 但如果新闻语境与市场语境匹配，并配合合适执行方式；
- 完全可能得到不错的交易展示效果。

#### 15.2 英文主线在因子层面更强

V3 regression 表明：

- 英文全球新闻在预测层面更有潜力；
- 尤其在回归 IC 上，明显优于 V1。

#### 15.3 高 IC 不等于高收益

这是整个项目最重要的经验之一：

- 单品种；
- 强 benchmark；
- 高频多空切换；
- 大波动周判断错误；

这些因素都会使“好因子”无法直接变成“高收益策略”。

#### 15.4 交易映射的重要性非常高

同一套模型：

- `always_in` 可能很差；
- `long_only` 可能立刻改善；
- 回归因子也会因为映射方式不同，表现完全不同。

#### 15.5 简单中英混合不一定有效

V2 的意义不在结果，而在提醒我们：

- 多语言不自动带来增强；
- 新闻源一致性、覆盖结构和质量比“新闻数量”更关键。

#### 15.6 成本与工程约束是真实问题

我们还发现：

- 国内大模型对英文长文本打分更慢；
- 英文打分成本明显高于中文；
- 工程稳定性、断点续跑和并行打分会显著影响研究效率。

---

### 16. 改进方向

未来可从三层继续改进：

#### 16.1 数据层

- 扩更多高质量英文能源专门新闻源
- 扩更多中文财经与能源新闻源
- 继续提升历史覆盖完整度
- 尝试日频甚至日内信号

#### 16.2 模型层

- 引入更高阶海外 LLM
- 恢复原论文的多模型组合
- 加入 FinBERT 类金融专用模型
- 比较不同大模型在五维打分上的系统偏差

#### 16.3 交易层

- 使用更真实的执行框架
- 优化单品种强 beta 下的仓位映射
- 做多品种因子组合
- 将新闻因子纳入更完整的因子库中检验互补性和共线性

---

### 17. 总结与最终口径

如果要把本项目浓缩成一句话，可以表述为：

**我们复现并迁移了原论文的多维 LLM 新闻因子框架，在中国 INE SC 原油期货上验证了新闻因子在预测层面和交易层面的条件性有效性；最终发现，中文单源分类更适合做直观展示，而英文主线回归更适合作为量化因子扩展方向。**

最终最推荐保留的两条主线是：

1. `V1 classification / long_only`
   - 适合课堂汇报
   - 直观、易解释、结果也好

2. `V3 regression / long_only_factor`
   - 适合强调“量化因子扩展”
   - 更能代表未来研究方向
